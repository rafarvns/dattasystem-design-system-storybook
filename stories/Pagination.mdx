# Pagination

O componente **Pagination** permite ao usuário navegar entre grandes conjuntos de dados divididos em páginas. É essencial para tabelas e listagens, garantindo que o usuário possa localizar registros sem sobrecarga cognitiva ou de performance.

## Quando usar
- Em tabelas com muitos registros (ex: processos, listagem de servidores).
- Em resultados de busca.
- Sempre que o volume de dados puder degradar o tempo de carregamento da página se exibido de uma só vez.

## Anatomia
O componente é composto por:
1. **Informação de resultados:** Exibe o intervalo atual e o total de itens.
2. **Controles de navegação:** Botões Anterior e Próximo.
3. **Botões numéricos:** Atalhos para acesso direto às páginas, com truncamento inteligente ("...") para lidar com grandes quantidades.
4. **Itens por página (opcional):** Seletor para alterar a densidade de registros.

## Acessibilidade
- Estruturado com `<nav>` e `aria-label="Paginação de resultados"`.
- Utiliza `<ul>` e `<li>` para listar os itens de navegação.
- A página atual é marcada com `aria-current="page"`.
- Botões desabilitados (ex: Anterior na primeira página) utilizam o atributo `disabled`.
- O ellipsis ("...") é ocultado de tecnologias assistivas via `aria-hidden="true"`.
- Navegação completa via teclado utilizando a tecla `Tab`.
