# Select

O componente **Select** permite que o usuário escolha uma única opção de uma lista predefinida em um menu suspenso. No Datta System, utilizamos o elemento `<select>` nativo para garantir a melhor acessibilidade e performance, especialmente em dispositivos de baixa especificação.

## Quando usar
- Quando houver muitas opções (mais de 4 ou 5) que tornariam o uso de Radio Buttons inviável.
- Quando o espaço na tela for limitado.
- Para seleções padrão como Estados, Cidades, Categorias, etc.

## Quando não usar
- Quando houver apenas 2 ou 3 opções (prefira **Radio Buttons** para maior visibilidade).
- Se a lista de opções for muito longa e exigir busca (considere um componente de Select com busca ou Autocomplete no futuro).

## Acessibilidade

O componente Select foi projetado seguindo as diretrizes WCAG 2.1 AA:
- **Nativo:** Utiliza o elemento `<select>` HTML5, garantindo suporte nativo a leitores de tela e navegação por teclado.
- **Rótulos:** A `<label>` está corretamente associada ao campo via atributo `for`/`id`.
- **Estados:** Atributos `aria-invalid`, `aria-required` e `aria-describedby` são gerenciados dinamicamente.
- **Navegação:** Suporte total a `Tab` para foco, `Enter`/`Space` para abrir e setas do teclado para navegar entre opções.

## Anatomia
O Select é composto por uma label superior, um wrapper que contém o elemento `<select>` nativo e um ícone de chevron customizado à direita, e uma área inferior para mensagens de ajuda ou erro.

## Tokens Utilizados
- **Cores:** `--color-gray-300` (borda), `--color-blue-500` (foco/ícone), `--color-red-500` (erro), `--color-white` (fundo).
- **Tipografia:** `--font-family-primary`, `--font-size-sm` (label), `--font-size-base` (select).
- **Forma:** `--radius-md`.
- **Espaçamento:** `--spacing-1`, `--spacing-3`, `--spacing-8` (reserva para seta).
